# OrthoAlign — план работ

## Цель проекта

Демо-проект под вакансию Lucky Hunter (AI Fullstack для ортодонтической компании). Доказывает, что инженер понимает бизнес-боль (узкое место — техник по планированию) и умеет автоматизировать её на их стеке (Python+MySQL+Redis+Celery / React+TS / WebGL).

## Что показывает CTO за 2 минуты

1. Открывает страницу, выбирает демо-кейс или загружает свой STL.
2. ~3-5 секунд, появляется челюсть с цветовой разметкой по зубам (FDI-нумерация).
3. Кликает зуб, тащит TransformControls в новую позицию.
4. Слайдер 0→20 — видит интерполяцию стадий лечения.
5. (v2) Кнопка «AI suggest» — Claude API предлагает движения.

## Архитектурные решения (зафиксированы)

- **r3f вместо голого Three.js** — DX в React несравнимо лучше.
- **Vite вместо Next.js** — SSR не нужен, нужна скорость dev-цикла.
- **JSON формат кейсов в v1, glTF в v2** — JSON проще на старте, glTF в 5-10 раз меньше для тех же данных.
- **MySQL+Redis+Celery в docker-compose, даже если в v1 не используются логически** — зеркалит стек вакансии.
- **Precomputed labels в v1, живой ML в v2** — снимает риск с первой версии.

## Риски

1. **Pretrained веса для tooth segmentation могут быть закрыты.** План Б: дообучить лёгкий PointNet++ на 50 сэмплах из Teeth3DS на CPU за ночь. Без претензий на SOTA.
2. **Лицензия Teeth3DS — CC BY-NC-SA.** Подходит для портфолио с указанием источника, не подходит для коммерческого использования. В README явно указано.
3. **Производительность Three.js на больших мешах (200k+ треугольников).** Decimation через trimesh до 30-50k на бэке.
